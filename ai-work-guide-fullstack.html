0. AI 怎么动手？四个东西，一个比喻

比喻：你雇了个管家，家里有各种电器（飞书、邮件、日历）。管家需要四样东西才能操作电器。
名字比喻你做什么难度
⚡API电线——电器能跑靠它不用碰⭐⭐⭐
🔌MCP标准插座——所有电器插口一样配一段 JSON⭐⭐
🎮CLI遥控器——管家按按钮装一个命令行工具⭐
📖Skill说明书——告诉管家操作流程写一个 SKILL.md⭐

一句话：管家读说明书（Skill）→ 走到插座前（MCP）→ 按遥控器（CLI）→ 电器开了（背后是 API）。

小白行动指引

```
  • MCP：魔搭 MCP 广场（阿里 1000+ 服务）· 火山引擎 MCP（字节）· 腾讯云 MCP

# rss_fetcher.py
import feedparser, json, os
from datetime import datetime

RSS_SOURCES = {
    "36氪": "https://36kr.com/feed",
    "少数派": "https://sspai.com/feed",
    "量子位": "https://www.qbitai.com/feed",
    "虎嗅": "https://www.huxiu.com/rss/0.xml",
    "极客公园": "https://www.geekpark.net/rss",
    "InfoQ": "https://www.infoq.cn/feed",
    "机器之心": "https://www.jiqizhixin.com/rss",
    "晚点": "https://www.latepost.com/rss",
    "钛媒体": "https://www.tmtpost.com/rss.xml",
    "Hacker News": "https://hnrss.org/frontpage",
    "阮一峰": "https://www.ruanyifeng.com/blog/atom.xml",
    "V2EX": "https://www.v2ex.com/index.xml",
    "TechCrunch": "https://techcrunch.com/feed/",
    "The Verge": "https://www.theverge.com/rss/index.xml",
    "Wired": "https://www.wired.com/feed/rss",
}

def fetch_all(max_per_source=5):
    items = []
    for name, url in RSS_SOURCES.items():
        try:
            feed = feedparser.parse(url)
            for e in feed.entries[:max_per_source]:
                items.append({"source": name, "title": e.get("title",""), "url": e.get("link",""), "summary": e.get("summary","")[:300]})
        except: pass
    return items

if __name__ == "__main__":
    items = fetch_all()
    os.makedirs("data", exist_ok=True)
    json.dump(items, open("data/raw.json", "w", encoding="utf-8"), ensure_ascii=False, indent=2)
    print(f"共 {len(items)} 篇")

1.2 调 AI 总结 + 推送飞书

# main.py
import json, os
from dotenv import load_dotenv
from openai import OpenAI
import requests

load_dotenv()

def summarize(items):
    text = "\n".join(f"[{i['source']}] {i['title']}" for i in items[:20])
    client = OpenAI(base_url="https://api.deepseek.com", api_key=os.getenv("DEEPSEEK_API_KEY"))
    r = client.chat.completions.create(
        model="deepseek-chat",
        messages=[{"role":"system","content":"你是新闻编辑，整理成 5 个主题分类，每类 1-2 句话总结 + 1 个最值得看的链接。"},
                  {"role":"user","content":text}])
    return r.choices[0].message.content

def push(content):
    requests.post(os.getenv("FEISHU_WEBHOOK_URL"), json={
        "msg_type": "interactive",
        "card": {"header": {"title": {"tag": "plain_text", "content": "📰 今日热点"}},
                 "elements": [{"tag": "markdown", "content": content}]}})

if __name__ == "__main__":
    from rss_fetcher import fetch_all
    items = fetch_all()
    summary = summarize(items)
    push(summary)
    os.makedirs("data", exist_ok=True)
    json.dump({"summary": summary, "items": items}, open("data/report.json", "w", encoding="utf-8"), ensure_ascii=False, indent=2)
    print("✅ 完成")

2. AI 手脚实操

2.1 飞书 CLI 选题流水线

4 步从"选题报告"到"内容生产"：

```
① 创建多维表格（先聊结构再动手）
   "帮我把选题报告放进飞书多维表格，先不急着创建，先跟我聊清楚这个表的结构。"
   → AI 反过来问你需求 → 你回答 → AI 列出表结构 → 你确认 → AI 执行

② 数据进表
   AI 读飞书文档 → 按字段填表格
   选题名称 / 方向 / 优先级 / 报告链接 → 自动入库

③ 撰写笔记 + 划词评论改稿
   AI 写小红书笔记存飞书文档
   你在文档里划词评论："标题选最吸睛的 / 正文选方向一"
   💡 评论 = AI 能通过 CLI 读到
   告诉 AI "根据评论修改" → 自动执行

④ 建拍摄日程
   "明天上午 9 点拍摄" → AI 直接在飞书日历建日程
   详情 + 参考文档自动填好

核心洞察：能做到多少不取决于 AI 模型，而取决于厂商的 CLI 开放了多少能力。飞书 CLI 是目前开放最多、迭代最快的。

三步法探索新 CLI：看厂商介绍 → 看别人实践 → 装到 agent 先问再测。

2.2 麦当劳 MCP 下单

```
① 麦当劳 MCP 平台 → 登录 → 激活 TOKEN（授权码）
② 复制接入指南的 JSON 配置 → 换掉 TOKEN 占位
③ WorkBuddy → 左侧连接器 → 自定义 MCP → 粘贴 JSON → 保存 → 绿灯
④ "附近有麦当劳吗" → MCP 查询（不是网搜）
   "点 1+1 套餐" → 选门店 + 选套餐 → 下单 → 扫码支付

💡 MCP 好处：服务端别人写好，你只配一段 JSON。新电器插上标准插座就能用。

一个平台三种接入（以 Get 笔记为例）：MCP（插座）+ CLI（遥控器）+ API（电线）。入门首选 CLI——AI 友好、快。
```

8. Agent 从零制作 + AI 复利系统（社群分享）

社群实战经验。用本文框架统一——聊天 AI 设计 · 干活 AI 制造。

8.1 分工：聊天 AI vs 干活 AI

```
聊天 AI（DeepSeek / Kimi / 通义 / 元宝）
    ↓ 聊需求、拆结构、出提示词
干活 AI（WorkBuddy / Trae Solo / Qoder / 扣子）
    ↓ 跑提示词 → 生成壳子 → 改 Bug → 部署飞书

聊天 AI 负责"说清楚"，干活 AI 负责"做出来"。

8.2 Agent 制作全流程

```
① 跟聊天 AI 聊需求
   打开 DeepSeek/Kimi → "我要做一个什么样的 Agent"
   → 聊清楚每个细节 → 让它生成一段"给干活 AI 的提示词"

② 干活 AI 跑出壳子
   把提示词粘贴给 WorkBuddy/Trae Solo/Qoder
   → 干活 AI 自动生成第一版代码和配置
   → 这一版叫 Agent 的"壳子"——有骨架，还没灵魂

③ 自己买 API 给灵魂
   去平台申请 API Key（DeepSeek / Claude / 通义）
   → 把 Key 配到壳子的配置里
   → Agent 有了"大脑"——能自己调 AI 了

④ 整理文档 → 交给干活 AI 后续维护
   把跟聊天 AI 聊天的全部信息整理成文档
   → 给干活 AI："照着这个文档，修 Bug + 部署到飞书"
   → 干活 AI 接手后续，你只负责用

8.3 飞书部署优势

```
飞书里发一条命令 → 后台本地 Agent 启动运行
不用开电脑软件，不用连服务器——在飞书聊天框就能操控。

8.4 两种 Agent

类型新手建议
别人写好的（扣子市场 / 虾评 / GitHub）❌ 第一次别用——你学不会
自己用聊天 AI + 干活 AI 做的✅ 第一次必须自己做——走通一次，就能造无数个

8.5 看到好东西想复刻？四步法

```
① 结构拆解（聊天 AI）
   把项目信息全部扔给 DeepSeek/Kimi
   "我想 1:1 复刻这个，帮我拆结构"

② 理解流程（聊天 AI）
   一直问，直到搞懂每一环节在干嘛
   "刚开始我也不知道怎么接 API，就是一直问"

③ 直接上手（干活 AI）
   知道流程后就做。把理解的内容写成提示词给干活 AI
   不用等别人教——直接开始，边做边问

④ 总结复制（干活 AI）
   做完让干活 AI 把整个过程总结成文档
   → 下次碰见类似的东西，扔文档 → 批量复制
   核心原则：在哪步问 AI "怎么做的" → 就让 AI 当场沉淀

8.6 AI 复利系统

核心思想：每次跟 AI 聊完，都变成一个文档。日积月累，自动分类。

```
每天：聊完任何 AI → 让它沉淀成文档 → 投到"无脑存放"文件夹
每隔几天：让 AI 把文件夹里的文档分类 + 总结 + 提炼

但关键在这里：

```
第 1-3 层：AI 帮你沉淀（分类、归纳、总结）
第 4-8 层：必须你自己来（深度洞察、审美判断、创意连接）

AI 只能帮你处理信息，不能帮你建立审美。

进阶方向（社群朋友的思想雏形，尚未实现）：

```
沉淀够多之后 → 让 AI 分析"我的沉淀习惯"
→ 试着让 AI 模仿我的方式自动沉淀
→ 目前：1-3 层 AI 做，4-8 层人做
💡 这个系统不止能用在 AI 上。聊天记录、读书笔记、灵感碎片都可以投进去——在人生所有维度上复利。
```
